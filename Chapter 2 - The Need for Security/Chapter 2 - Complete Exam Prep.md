## üö® **THE 12 CATEGORIES OF THREATS**

### **1. Compromises to Intellectual Property**
- **Software Piracy**: 39% of software globally unlicensed ($52.2 billion value)
- **Protection Methods**: Digital watermarks, registration codes, EULAs, online registration
- **Watchdog Organizations**: SIIA and BSA

### **2. Deviations in Quality of Service**
- **Service Level Agreement (SLA)**: Specifies expected service levels and availability
- **Types**: ISP failures, power outages, telecommunications issues
- **Cost Example**: 99.9% uptime downtime costs $10,950 average

### **3. Espionage or Trespass**
- **Competitive Intelligence**: Legal collection of competitor information
- **Industrial Espionage**: Illegal collection for unfair advantage
- **Shoulder Surfing**: Direct observation of information entry
- **Hacker Types**: Expert (high skill), Novice (limited skill), Professional (financial motive), Penetration Testers (authorized)

### **4. Forces of Nature**
- **Types**: Fire, floods, earthquakes, lightning, tornados, hurricanes, tsunamis
- **Special Considerations**: ESD (10 volts can damage electronics), dust contamination, solar activity
- **Mitigation**: Insurance, physical protection, geographic considerations

### **5. Human Error or Failure** (GREATEST THREAT)
- **Examples**: Accidental disclosure, incorrect data entry, failure to follow procedures
- **Famous Cases**: 1997 Internet router error (45% of users affected)
- **Prevention**: Training, dual-approval controls, expert systems monitoring

### **6. Information Extortion**
- **Ransomware Types**: Lockscreen (blocks access), Encryption (encrypts files)
- **Famous Cases**: CD Universe ($100,000 demand), Express Scripts, Colonial Pipeline
- **Response**: Never pay ransoms, focus on prevention and recovery

### **7. Sabotage or Vandalism**
- **Hacktivism/Cyberactivism**: Using hacking to protest organizations
- **Cyberterrorism**: Politically motivated attacks against information systems
- **Cyberwarfare**: Government-sanctioned offensive operations

### **8. Software Attacks (Malware)**
- **Virus**: Code attached to programs that replicates and spreads
- **Worm**: Self-replicating malware (MyDoom: $38 billion damage)
- **Trojan Horse**: Malicious software disguised as legitimate
- **Spyware/Adware**: Secret information gathering or unwanted advertising

### **9. Technical Hardware Failures**
- **Famous Cases**: Intel Pentium FDIV bug ($475 million loss)
- **MTBF**: Mean Time Between Failure measurements
- **Hard Drives**: Most commonly failing component (500,000 hours average MTBF)

### **10. Technical Software Failures**
- **OWASP Top 10**: Injection, broken authentication, XSS, insecure references, misconfiguration, data exposure, missing access control, CSRF, vulnerable components, unvalidated redirects
- **24 Deadly Sins**: Buffer overruns, XSS, SQL injection, improper error handling, weak passwords, etc.

### **11. Technological Obsolescence**
- **Examples**: Windows XP end-of-support (2014), Symantec software retirement
- **Impact**: Loss of vendor support, increased vulnerabilities, higher costs

### **12. Theft**
- **Types**: Physical (hardware), electronic (data), intellectual property
- **Mobile Risk**: Smartphones/tablets increase theft risk, stored credentials valuable

## üîê **PASSWORD SECURITY**

### **Password Attack Types**
1. **Brute Force**: Trying every possible combination
2. **Dictionary**: Using common passwords and personal info
3. **Rainbow Tables**: Using precomputed hash tables
4. **Social Engineering**: Tricking users into revealing passwords

### **Password Strength Analysis**
- **8-character case-insensitive**: 1.01 seconds to crack
- **8-character with numbers/symbols**: 2.7 hours to crack
- **12-character with numbers/symbols**: 14,141.9 years to crack

### **10.4 Password Rule**
- 10+ characters, 1 uppercase, 1 lowercase, 1 number, 1 special character

## üìù **SHORT ANSWER QUESTIONS**

### Question 1
List and briefly explain the four important functions that information security performs for an organization.

**Answer:**
1. **Protecting the Organization's Ability to Function**: Ensures business continuity and protects against operational disruptions
2. **Protecting the Data Organizations Collect and Use**: Safeguards critical business data and information
3. **Enabling the Safe Operation of Applications**: Protects infrastructure applications that support business operations
4. **Safeguarding Technology Assets**: Protects hardware and software investments that support information systems

### Question 2
What is the difference between a threat and an attack in the context of information security?

**Answer:**
- **Threat**: Any event or circumstance that has the potential to adversely affect operations and assets. Threats are always present and represent potential risks
- **Attack**: An intentional or unintentional act that can damage or otherwise compromise information and the systems that support it. Attacks exist only when a specific act may cause a loss

### Question 3
Describe the difference between a virus, a worm, and a Trojan horse.

**Answer:**
- **Virus**: Code segments that attach to existing programs and execute when the host program runs
- **Worm**: Self-replicating malware that can spread independently without attaching to programs
- **Trojan Horse**: Malicious software disguised as legitimate, helpful, or necessary programs

### Question 4
What are the two types of ransomware, and how do they differ?

**Answer:**
1. **Lockscreen Ransomware**: Denies access to the user's system by disabling access to the desktop
2. **Encryption Ransomware**: Encrypts some or all of a user's hard drive and demands payment for the decryption key

### Question 5
Explain what a Service Level Agreement (SLA) is and why it's important for information security.

**Answer:**
**Service Level Agreement (SLA)**: A document that specifies the expected level of service from a service provider, including provisions for minimum acceptable availability and penalties or remediation procedures for downtime.

**Importance**: Ensures service availability, helps maintain business continuity, provides framework for addressing service interruptions, and may include penalties for service provider failures.

### Question 6
Explain the concept of "shoulder surfing" and provide examples of where it commonly occurs.

**Answer:**
**Shoulder surfing** is the direct, covert observation of individual information or system use, typically occurring in public or semipublic settings.

**Common locations:**
- Computer terminals and desks in public areas
- ATMs and banking kiosks
- Public transportation (buses, airplanes, subways) where people use smartphones and tablets
- Coffee shops and public Wi-Fi areas
- Open office environments

**Professional etiquette**: If you can see another person entering personal or private information, you should look away as the information is entered.

### Question 7
What are the three main types of hackers, and how do their skill levels and motivations differ?

**Answer:**
1. **Expert Hackers**: 
   - High technical skill level
   - Master multiple programming languages and systems
   - Often create automated exploits and tools
   - May become dissatisfied with direct attacks and turn to writing software

2. **Novice Hackers**:
   - Limited technical skills
   - Use tools created by expert hackers
   - Include script kiddies and packet monkeys
   - Often caught due to poor operational security

3. **Professional Hackers**:
   - Conduct attacks for personal financial benefit
   - May work for crime organizations or foreign governments
   - Should not be confused with penetration testers
   - Motivated by profit rather than technical challenge

### Question 8
Describe the difference between a virus, a worm, and a Trojan horse.

**Answer:**
- **Virus**: Code segments that perform malicious actions and attach themselves to existing programs. They replicate and propagate when the host program is executed, often spreading through email attachments or infected files.

- **Worm**: Self-replicating malware that can activate and replicate without being attached to an existing program. Named after tapeworms, they can fill available resources until systems become unresponsive.

- **Trojan Horse**: Malicious software disguised as legitimate, helpful, or necessary programs. Like the Greek legend, once brought into a system, they become activated and can wreak havoc on unsuspecting users.

### Question 9
What is software piracy, and what are the main methods used to combat it?

**Answer:**
**Software Piracy**: The unauthorized duplication, installation, or distribution of copyrighted computer software, which is a violation of intellectual property.

**Methods to combat piracy:**
- **Technical mechanisms**: Digital watermarks, embedded code, copyright codes, intentional bad sectors
- **Software registration codes**: Unique codes combined with end-user license agreements (EULAs)
- **Online registration**: Required registration during installation or for technical support
- **Legal enforcement**: Laws against software piracy and watchdog organizations
- **Watchdog organizations**: Software & Information Industry Association (SIIA) and Business Software Alliance (BSA)

### Question 10
What are the two types of ransomware, and how do they differ in their approach?

**Answer:**
1. **Lockscreen Ransomware**:
   - Denies access to the user's system by disabling access to the desktop
   - Prevents users from bypassing the ransom screen
   - Less sophisticated but still effective at blocking access

2. **Encryption Ransomware**:
   - Encrypts some or all of a user's hard drive
   - Demands payment for the decryption key
   - More sophisticated and potentially more damaging
   - May permanently destroy data if payment is not made

### Question 11
List five of the 24 "deadly sins" in software security and explain why each is problematic.

**Answer:**
1. **Buffer Overruns**: Occur when more data is sent to a program buffer than it can handle, potentially allowing attackers to execute malicious code or cause system crashes.

2. **Cross-Site Scripting (XSS)**: Allows attackers to inject malicious scripts into web applications, potentially stealing user credentials or session information.

3. **SQL Injection**: Occurs when user input is not properly validated before being used in database queries, allowing attackers to access or manipulate database contents.

4. **Failure to Handle Errors**: Poor error handling can cause unexpected system behavior, crashes, or information disclosure that could be exploited by attackers.

5. **Use of Weak Password-Based Systems**: Insufficient password strength requirements and poor password management make systems vulnerable to brute force and dictionary attacks.

### Question 12
What is the difference between a denial-of-service (DoS) attack and a distributed denial-of-service (DDoS) attack?

**Answer:**
- **Denial-of-Service (DoS)**: A single attacker sends a large number of connection or information requests to a target, overwhelming the system's ability to respond to legitimate requests.

- **Distributed Denial-of-Service (DDoS)**: A coordinated stream of requests is launched against a target from many locations simultaneously using compromised machines (bots or zombies) that are remotely controlled by the attacker.

**Key differences:**
- **Source**: DoS comes from one source; DDoS comes from multiple sources
- **Complexity**: DDoS is more complex to execute and defend against
- **Scale**: DDoS can generate much larger volumes of traffic
- **Detection**: DDoS attacks are harder to block because they come from multiple legitimate-looking sources

### Question 13
Explain the concept of "privilege escalation" and provide examples of how it can occur.

**Answer:**
**Privilege Escalation**: The unauthorized modification of an authorized or unauthorized system user account to gain advanced access and control over system resources.

**Examples:**
- **Jailbreaking**: Gaining administrator-level control over iOS devices, allowing installation of unauthorized applications
- **Rooting**: Gaining administrator-level control over Android devices
- **Back doors**: Installing unauthorized access mechanisms in compromised systems
- **Exploiting vulnerabilities**: Using software flaws to gain higher privileges than originally assigned
- **Social engineering**: Tricking administrators into granting higher privileges

### Question 14
What is the "10.4 password rule" and why is it important for password security?

**Answer:**
**The 10.4 Password Rule**:
- At least **10** characters long
- At least **1** uppercase letter
- At least **1** lowercase letter  
- At least **1** number
- At least **1** special character

**Importance:**
- **Exponential security increase**: Each additional character type dramatically increases the password space
- **Resistance to brute force**: Makes brute force attacks computationally infeasible
- **Dictionary attack resistance**: Reduces effectiveness of dictionary-based attacks
- **Industry best practice**: Widely accepted standard for password strength
- **Regulatory compliance**: Meets many industry and government security requirements

### Question 15
Describe the difference between phishing, spear phishing, and whaling attacks.

**Answer:**
- **Phishing**: Broad-based fraudulent emails sent to many recipients, typically using the names of large banks or retailers to trick users into revealing sensitive information.

- **Spear Phishing**: Highly targeted phishing attacks sent to a small group or even one person, appearing to come from an employer, colleague, or other legitimate correspondent.

- **Whaling**: A form of spear phishing specifically targeting senior executives or high-value individuals within an organization.

**Key differences:**
- **Target scope**: Phishing targets many; spear phishing targets few; whaling targets specific high-value individuals
- **Personalization**: Phishing is generic; spear phishing is personalized; whaling is highly customized
- **Sophistication**: Each type increases in sophistication and potential impact

### Question 1
Compare and contrast the different types of malware. Discuss their characteristics, methods of propagation, and potential impact on organizations.

**Answer:**

**VIRUSES**
- **Characteristics**: Code segments that attach to existing programs and execute when the host program runs
- **Propagation**: Through infected files, email attachments, removable media, and network shares
- **Impact**: Can corrupt files, delete data, display messages, or perform other malicious actions
- **Example**: ILOVEYOU virus (2000) - infected 10% of Internet users, caused $5.5 billion damage

**WORMS**
- **Characteristics**: Self-replicating malware that can spread independently without attaching to programs
- **Propagation**: Through network vulnerabilities, email, instant messaging, and file sharing
- **Impact**: Can consume system resources, slow networks, and create backdoors for attackers
- **Example**: MyDoom worm (2004) - infected 2 million systems, caused $38 billion damage

**TROJAN HORSES**
- **Characteristics**: Malicious software disguised as legitimate programs or files
- **Propagation**: Through email attachments, software downloads, and social engineering
- **Impact**: Can install backdoors, steal information, or perform other unauthorized actions
- **Example**: Happy99.exe - disguised as a New Year's celebration program while installing backdoor

### Question 2
Analyze the role of human error in information security. Why is it considered the greatest threat? Discuss specific examples and explain what organizations can do to mitigate this threat.

**Answer:**

**Why Human Error is the Greatest Threat:**
- **Proximity to information**: Employees are closest to the data and systems they use daily
- **Frequency of occurrence**: Human errors happen constantly in all organizations
- **Difficulty of prevention**: Unlike technical threats, human behavior is harder to control
- **Compounding effects**: Human errors can create vulnerabilities that technical threats exploit

**Specific Examples:**
- **1997 Internet Router Error**: Simple keyboarding error caused major Internet outage affecting 45% of users
- **Metro-North Power Loss (2014)**: Repair technicians failed to note system interconnections
- **Common Daily Errors**: Leaving classified information unprotected, using weak passwords, clicking suspicious attachments

**Mitigation Strategies:**
1. **Comprehensive Training**: Regular security awareness training and role-specific education
2. **Clear Procedures**: Simple, well-documented procedures with regular review
3. **Technical Controls**: Dual-approval systems, automated monitoring, expert systems
4. **Usability Design**: Making secure actions the default and easiest option
5. **Organizational Culture**: Leadership commitment, encouraging reporting, learning from mistakes

### Question 3
Examine the concept of information extortion and ransomware. How have these threats evolved, and what makes them particularly dangerous? Discuss prevention strategies and response measures.

**Answer:**

**Evolution of Information Extortion and Ransomware:**

**Early Forms (1990s-2000s):**
- **CD Universe Case**: Russian hacker Maxus stole credit card numbers and demanded $100,000
- **Express Scripts**: Hacker demonstrated access to 75 customer records, claimed millions more
- **Simple threats**: Basic blackmail attempts using stolen data

**Modern Ransomware (2010s-Present):**
- **Sophisticated encryption**: Advanced cryptographic techniques
- **Automated distribution**: Large-scale campaigns through email and web attacks
- **Ransom-as-a-Service**: Criminals offering ransomware tools to other criminals
- **Targeted attacks**: Focused on high-value organizations like hospitals and government agencies

**What Makes Ransomware Particularly Dangerous:**
1. **Immediate Impact**: Organizations can lose access to critical data instantly
2. **High Success Rate**: Even sophisticated organizations can fall victim
3. **Low Technical Barrier**: Attackers can use pre-built tools
4. **Difficult Recovery**: Without backups, data may be permanently lost
5. **Business Disruption**: Can shut down operations for days or weeks
6. **Legal and Regulatory Issues**: May trigger breach notification requirements

**Prevention Strategies:**
1. **Technical Controls**: Regular backups, email filtering, web filtering, application whitelisting, system updates, network segmentation
2. **User Education**: Security awareness training, simulated attacks, reporting procedures
3. **Organizational Measures**: Incident response planning, business continuity planning, vendor management, insurance coverage

**Response Measures:**
1. **Immediate Response**: Isolate affected systems, assess scope, activate incident response team, notify stakeholders
2. **Recovery Operations**: Restore from backups, rebuild systems, validate data, implement security hardening
3. **Post-Incident Activities**: Forensic analysis, lessons learned, communication, legal considerations

**Best Practices:**
- **Never pay ransoms**: This encourages further attacks and doesn't guarantee data recovery
- **Focus on prevention**: Invest in robust backup and security systems
- **Regular testing**: Test backup and recovery procedures regularly
- **Continuous monitoring**: Implement security monitoring and alerting systems

### Question 6
Analyze the OWASP Top 10 web application security risks. How do these risks relate to the broader categories of threats discussed in the chapter? What steps should organizations take to address these vulnerabilities?

**Answer:**

**OWASP Top 10 (2013) Analysis:**

**1. Injection**: Directly relates to **Technical Software Failures** - input validation vulnerabilities
**2. Broken Authentication and Session Management**: Relates to **Technical Software Failures** and **Human Error** - weak authentication mechanisms
**3. Cross-Site Scripting (XSS)**: **Technical Software Failures** - client-side input validation issues
**4. Insecure Direct Object References**: **Technical Software Failures** - authorization vulnerabilities
**5. Security Misconfiguration**: **Human Error** and **Technical Software Failures** - improper system configuration
**6. Sensitive Data Exposure**: **Technical Software Failures** and **Human Error** - inadequate data protection
**7. Missing Function Level Access Control**: **Technical Software Failures** - authorization bypass vulnerabilities
**8. Cross-Site Request Forgery (CSRF)**: **Technical Software Failures** - session management issues
**9. Using Components with Known Vulnerabilities**: **Technical Software Failures** - supply chain vulnerabilities
**10. Unvalidated Redirects and Forwards**: **Technical Software Failures** - input validation issues

**Relationship to Broader Threat Categories:**

**Primary Category - Technical Software Failures:**
- Most OWASP Top 10 items are software vulnerabilities
- Result from poor development practices and inadequate testing
- Can be exploited by various attack vectors

**Secondary Category - Human Error:**
- Configuration mistakes (Security Misconfiguration)
- Inadequate training of developers
- Poor security awareness in development teams

**Attack Vector - Software Attacks:**
- OWASP vulnerabilities are often exploited by malware and automated attacks
- Can be used as entry points for more sophisticated attacks

**Steps to Address OWASP Vulnerabilities:**

**Development Phase:**
- **Secure Coding Training**: Train developers on secure coding practices
- **Code Reviews**: Implement mandatory security code reviews
- **Static Analysis**: Use automated tools to detect vulnerabilities during development
- **Threat Modeling**: Include threat modeling in design phase

**Testing Phase:**
- **Security Testing**: Comprehensive security testing including penetration testing
- **Vulnerability Scanning**: Regular automated vulnerability assessments
- **Dynamic Analysis**: Runtime security testing and monitoring

**Deployment and Operations:**
- **Secure Configuration**: Implement secure default configurations
- **Regular Updates**: Keep all components and dependencies updated
- **Monitoring**: Continuous monitoring for security incidents
- **Incident Response**: Prepared response procedures for security incidents

**Organizational Measures:**
- **Security Policies**: Establish clear security policies for web applications
- **Training Programs**: Regular security awareness training for all staff
